GLOBAL TECH UNIVERSITY — COURSE SYLLABUS
CS330: CYBERSECURITY FUNDAMENTALS
Credits: 3 | Lecture: 3 hours/week
Department: Computer Science and Engineering
Prerequisites: CS210 (minimum grade C) AND CS230 (minimum grade C)

COURSE DESCRIPTION

CS330: Cybersecurity Fundamentals is a comprehensive introduction to the theory, practice, and ethics of computer security. As digital systems become increasingly integrated into the critical infrastructure of society — power grids, financial systems, healthcare, democratic elections, personal communication — the security of those systems becomes a matter of profound public concern. CS330 prepares students to think and work as security-minded engineers: to understand how systems are attacked, how they are defended, and how security properties can be built into systems from their design rather than bolted on as an afterthought.

The course is organized around five major topic areas. The first is cryptographic foundations: students study symmetric encryption (AES), asymmetric encryption (RSA, elliptic curve cryptography), cryptographic hash functions, message authentication codes, digital signatures, and public key infrastructure. The second is network security: students examine common network attacks (ARP spoofing, DNS poisoning, man-in-the-middle attacks, DDoS) and defenses (TLS, IPsec, firewalls, intrusion detection). The third is systems security: students study memory safety vulnerabilities (buffer overflows, format string attacks, use-after-free), privilege escalation, operating system security mechanisms, and secure software development practices. The fourth is web security: students study the OWASP Top 10 in depth, including SQL injection, cross-site scripting, cross-site request forgery, and authentication vulnerabilities. The fifth is security policy, ethics, and law: students examine security from an organizational and societal perspective, including threat modeling, security audits, responsible disclosure, and the legal framework governing computer security.

The course includes a hands-on lab component using a capture-the-flag (CTF) format, in which students practice offensive security techniques in controlled environments to develop the attacker's perspective that is essential for effective defensive security work.

LEARNING OBJECTIVES

Explain the CIA triad (confidentiality, integrity, availability) and apply it as a framework for security analysis.
Describe and apply symmetric encryption algorithms (AES in CBC and GCM modes) and explain the importance of proper key management.
Describe the RSA algorithm mathematically and explain how its security depends on the difficulty of integer factorization.
Explain elliptic curve cryptography at a conceptual level and compare it to RSA in terms of key size and security level.
Compute and verify cryptographic hash functions (SHA-256, SHA-3), explain the avalanche effect and collision resistance, and apply Merkle trees.
Implement and explain HMAC, digital signatures, and public key infrastructure (PKI), including certificate chains and the role of certificate authorities.
Explain and demonstrate common network attacks: ARP spoofing, DNS cache poisoning, TCP session hijacking, and SYN flood DDoS.
Explain the TLS 1.3 handshake protocol in detail and identify common TLS misconfigurations.
Explain buffer overflow vulnerabilities, stack smashing, and return-oriented programming (ROP) attacks at a technical level.
Exploit a buffer overflow vulnerability in a controlled CTF environment and explain the mitigations (ASLR, stack canaries, NX bit) that prevent it in hardened systems.
Identify and exploit SQL injection, XSS, CSRF, and IDOR vulnerabilities in a controlled environment.
Conduct a threat model for a given system using the STRIDE methodology.
Apply secure coding practices in C and Python to prevent common vulnerability classes.
Describe the legal and ethical framework governing security research, penetration testing, and vulnerability disclosure.
Write a professional security audit report documenting identified vulnerabilities, their severity (CVSS scoring), and recommended remediations.

WEEKLY COURSE BREAKDOWN

Week 1: Introduction — The Cybersecurity Landscape, the CIA Triad, and the Attacker's Mindset
Week 2: Cryptography I — Symmetric Encryption (AES), Block Cipher Modes
Week 3: Cryptography II — Public Key Cryptography (RSA, ECC), Key Exchange (Diffie-Hellman)
Week 4: Cryptography III — Hash Functions, MACs, Digital Signatures, and PKI
Week 5: Network Security I — Network Attack Techniques (ARP, DNS, TCP Attacks)
Week 6: Network Security II — TLS/SSL, IPsec, VPNs, Firewalls, and IDS/IPS
Week 7: Systems Security I — Memory Safety Vulnerabilities (Buffer Overflow, Format String)
Week 8: Midterm Examination; Systems Security II — Advanced Exploitation (ROP, Heap Exploitation) and Mitigations (ASLR, Stack Canaries, NX)
Week 9: Web Security I — SQL Injection, XSS, CSRF, and Clickjacking (OWASP Top 10, Part 1)
Week 10: Web Security II — Authentication Vulnerabilities, IDOR, SSRF, XXE (OWASP Top 10, Part 2)
Week 11: Malware Analysis — Viruses, Worms, Ransomware, and Rootkits; Static and Dynamic Analysis
Week 12: Threat Modeling — STRIDE, Attack Trees, and Security Requirements Engineering
Week 13: Penetration Testing — Methodology, Tools (Metasploit, Burp Suite, Nmap), and CTF Lab
Week 14: Security Policy, Legal Framework (CFAA, GDPR), Responsible Disclosure, and Ethics
Week 15: Emerging Threats — Supply Chain Attacks, AI Security, Quantum Cryptography; Final CTF Competition and Course Synthesis


ACADEMIC CONTEXT

CS330 is among the most socially relevant courses in the GTU curriculum. The cybersecurity landscape of the 2020s is characterized by a scale and sophistication of threat that would have been unimaginable two decades ago: nation-state sponsored attacks on critical infrastructure, ransomware operations that have shut down hospitals and pipeline networks, massive data breaches exposing the personal information of hundreds of millions of people, and disinformation campaigns that exploit the algorithmic architecture of social media platforms. The engineers who design and build the digital systems that underlie modern civilization have a professional responsibility — and, GTU argues, a moral responsibility — to build those systems to be as secure as possible.

The prerequisite of CS210 (Operating Systems) and CS230 (Computer Networks) ensures that students arrive in CS330 with the technical foundation to engage with security at a mechanistic level. Understanding a buffer overflow requires understanding the stack frame structure of a process (CS210). Explaining an ARP spoofing attack requires understanding the ARP protocol and Ethernet frame structure (CS230). Security education that is not grounded in this kind of technical depth produces practitioners who can use security tools but cannot reason about them, modify them, or understand their limitations — which is not adequate preparation for the adversarial environment of real-world security practice.

The capture-the-flag (CTF) format of the lab sections is the gold standard in security education for developing the attacker's perspective that defensive security work requires. CTF challenges — finding hidden flags by exploiting vulnerabilities in intentionally vulnerable systems — provide a controlled, legal, and motivating environment in which students can practice offensive techniques that would be illegal if applied to real systems. The GTU CS department maintains a dedicated CTF lab environment (the GTU CyberArena) that is isolated from the university network and updated each semester with new challenges. Student teams regularly compete in external CTF competitions, where they consistently perform among the top entrants from undergraduate institutions.

The ethics and legal content of Week 14 is essential and non-negotiable. Security knowledge is inherently dual-use: the same skills used to conduct authorized penetration tests can be used to commit computer crimes. The Computer Fraud and Abuse Act (CFAA) in the United States, and analogous laws in other jurisdictions, makes unauthorized computer access a serious criminal offense, and students must understand where the legal boundaries lie. The responsible disclosure ethics discussion — covering the debate between full disclosure, coordinated disclosure, and bug bounty programs — introduces students to the professional norms of the security community and the ethical obligations that accompany security expertise.
