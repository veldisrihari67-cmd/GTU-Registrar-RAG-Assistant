GLOBAL TECH UNIVERSITY — COURSE SYLLABUS
CS102: DATA STRUCTURES
Credits: 3 | Lecture: 3 hours/week | Lab: See CS102L
Department: Computer Science and Engineering
Prerequisites: CS101 with minimum grade of C+
Co-requisite: CS102L (Data Structures Lab)

COURSE DESCRIPTION

CS102: Data Structures is the second core course in the Computer Science program at Global Tech University and represents a significant step up in both conceptual depth and programming sophistication from CS101. Whereas CS101 focused on learning to program, CS102 focuses on learning to program well — specifically, on choosing and implementing the right data organization for the problem at hand.
A data structure is a way of organizing, storing, and managing data in a computer so that it can be accessed and modified efficiently. The choice of data structure is one of the most consequential decisions a programmer makes: the difference between an array and a linked list, between a hash table and a balanced binary search tree, can determine whether a program runs in milliseconds or hours. This course introduces students to the foundational data structures used throughout computing — arrays, linked lists, stacks, queues, trees, heaps, hash tables, and graphs — and analyzes each in terms of its operational characteristics, use cases, and limitations.
Students will implement these data structures from scratch in Python and Java, developing a dual-language fluency that prepares them for the diversity of programming environments they will encounter in professional practice. The course also introduces Big-O notation and asymptotic analysis, equipping students to reason formally about the time and space complexity of their implementations. By the end of CS102, students will understand not just how to use data structures, but how they work internally, why they were designed the way they were, and when each is the appropriate choice.

LEARNING OBJECTIVES

Upon successful completion of CS102, students will be able to:

Explain the concept of abstract data types (ADTs) and distinguish between an ADT's interface and its implementation.
Define and apply Big-O notation to describe the time and space complexity of algorithms and data structure operations.
Implement and analyze singly linked lists, doubly linked lists, and circular linked lists from scratch.
Implement stack and queue ADTs using both array-based and linked-list-based approaches, and explain the trade-offs.
Implement and traverse binary trees, including binary search trees (BST), and perform insertions, deletions, and searches.
Explain and implement tree balancing concepts using AVL trees or red-black trees at a conceptual level.
Implement heap data structures and apply them to the priority queue ADT.
Implement hash tables using separate chaining and open addressing, and analyze hash function quality.
Represent graphs using adjacency matrices and adjacency lists and explain the trade-offs.
Implement fundamental graph traversal algorithms: breadth-first search (BFS) and depth-first search (DFS).
Apply recursion as a problem-solving technique to tree and graph problems.
Analyze and compare sorting algorithms (selection sort, insertion sort, merge sort, quicksort, heap sort) in terms of time complexity and stability.
Apply appropriate data structures to solve novel programming problems, justifying choices based on complexity analysis.
Write Java programs demonstrating understanding of classes, interfaces, generics, and the Java Collections Framework.
Test data structure implementations using systematic test cases, including edge cases and performance benchmarks.
Read and produce technical documentation describing data structure specifications and API design.

WEEKLY COURSE BREAKDOWN

Week 1: Review and Transition — From Python to Java
Topics: CS101 review (Python), motivation for CS102, introduction to Java syntax, compiling and running Java programs, primitive types vs. reference types, classes and objects in Java.
Assignments: Java warm-up exercises; setup lab.
Week 2: Abstract Data Types and Complexity Analysis
Topics: What is an ADT, interfaces in Java, Big-O notation, omega and theta notation, analyzing loops and nested loops, best/worst/average case analysis, space complexity.
Assignments: Problem set 1 — complexity analysis exercises.
Week 3: Arrays and Dynamic Arrays
Topics: Static arrays in Java, array manipulation, the ArrayList class, dynamic resizing and amortized analysis, array-based implementation of stacks and queues, array vs. linked structure trade-offs.
Assignments: Problem set 2 — array-based implementations.
Week 4: Linked Lists I — Singly Linked Lists
Topics: Node structure, head/tail pointers, traversal, insertion at head/tail/middle, deletion, search, linked list vs. array comparison, implementing a singly linked list in Java.
Assignments: Problem set 3 — singly linked list implementation; Quiz 1.
Week 5: Linked Lists II — Doubly and Circular Linked Lists
Topics: Doubly linked list structure, bidirectional traversal, sentinel nodes, circular linked lists, implementing the Deque ADT using a doubly linked list.
Assignments: Problem set 4 — doubly linked list; Lab project 1 (linked list application).
Week 6: Stacks and Queues
Topics: Stack ADT (push, pop, peek, isEmpty), applications of stacks (expression evaluation, backtracking, function call stack), Queue ADT, circular queue, deque ADT, priority queue introduction.
Assignments: Problem set 5 — stack/queue applications; Midterm Project Announced.
Week 7: Recursion
Topics: The recursion principle, base cases and recursive cases, call stack visualization, recursion vs. iteration, tail recursion, recursive data structures, common pitfalls (infinite recursion, stack overflow).
Assignments: Problem set 6 — recursive algorithms; Quiz 2.
Week 8: Midterm Examination and Review
Topics: Comprehensive review Weeks 1–7; Midterm Exam; Midterm Project due (implement a text-based to-do list manager using linked list structures).
Assignments: Midterm Exam; Midterm Project submission.
Week 9: Binary Trees and Binary Search Trees
Topics: Tree terminology, binary tree definition, traversal methods (in-order, pre-order, post-order, level-order), BST property, BST operations (search, insert, delete), BST performance analysis.
Assignments: Problem set 7 — BST implementation.
Week 10: Balanced Trees and Heaps
Topics: Motivation for balanced trees, BST degeneration, AVL tree rotations (conceptual), heap property, min-heap and max-heap, heap operations (insert, extract-min/max, heapify), the priority queue.
Assignments: Problem set 8 — heap implementation; Lab project 2 (priority queue scheduler).
Week 11: Hash Tables
Topics: Hash functions, collision resolution (separate chaining, linear probing, quadratic probing, double hashing), load factor, rehashing, hash table performance analysis, Java's HashMap and HashSet.
Assignments: Problem set 9 — hash table implementation; Quiz 3.
Week 12: Graphs — Representation and Traversal
Topics: Graph terminology (vertices, edges, directed/undirected, weighted, degree), adjacency matrix, adjacency list, BFS algorithm and applications, DFS algorithm and applications, cycle detection.
Assignments: Problem set 10 — graph representation and traversal.
Week 13: Sorting Algorithms
Topics: Selection sort, insertion sort, merge sort (divide and conquer), quicksort (pivot selection, partitioning), heap sort, counting sort and radix sort, stability, in-place vs. out-of-place, practical performance.
Assignments: Problem set 11 — sorting algorithm comparison and benchmarking; Final Project Announced.
Week 14: Advanced Topics and Integration
Topics: Tries (prefix trees), disjoint sets (union-find), algorithm design patterns revisited (greedy introduction, dynamic programming preview), Java Generics and the Java Collections Framework in depth.
Assignments: Problem set 12 — advanced structures; Final Project work session.
Week 15: Final Project Presentations and Course Wrap-Up
Topics: Final project presentations (a fully functional application using at least three distinct data structures, with written performance analysis), course retrospective, preview of CS201 (Algorithms).
Assignments: Final Project submission and presentation.

ACADEMIC CONTEXT

CS102 is, by virtually universal consensus among computer science educators, one of the two or three most important courses in the undergraduate CS curriculum. The data structures introduced in this course — linked lists, trees, hash tables, graphs — are not historical curiosities or textbook abstractions. They are the living infrastructure of every significant software system in existence. The HashMap that powers Python dictionaries, the heap that drives Dijkstra's shortest-path algorithm in your GPS application, the graph that represents the social network connecting billions of people — all of these are direct descendants of the structures studied in CS102.
The prerequisite of C+ in CS101 reflects the faculty's carefully considered judgment that CS102 is not a course where marginal programming fluency can be compensated by intellectual effort alone. The implementations required in CS102 are technically demanding: students will write Java code that manages memory explicitly through object references, constructs recursive solutions to tree problems, and analyzes the asymptotic behavior of their own code. A student who left CS101 still struggling with functions, loops, and basic data manipulation will find CS102 overwhelming. The C+ threshold is set not to exclude students, but to protect them from an experience that could undermine their confidence and derail their academic trajectory.
The introduction of Java alongside Python in CS102 is pedagogically intentional. Python's dynamic typing and garbage collection shield the programmer from many of the low-level realities that Java exposes. By learning to implement data structures in Java — with its explicit type declarations, interface-based design, and emphasis on object-oriented architecture — students encounter these realities directly. They learn what it means for a data structure to be generic (parameterized by type), what it means for an interface to specify a contract that multiple implementations must fulfill, and what it means for the compiler to enforce a set of structural rules at the time of program creation rather than at runtime. This dual-language approach also begins building the linguistic flexibility that is essential for professional software engineers, who routinely work in multiple languages and must be able to read and reason about code in any of them.
The Big-O notation introduced in Week 2 and applied throughout the remainder of the course is one of the most valuable intellectual tools a computer scientist possesses. It provides a language for comparing algorithms and data structures that transcends the specifics of hardware, operating systems, and programming languages. When a student learns that a hash table lookup is O(1) while a binary search tree lookup is O(log n), and that both are vastly superior to a linear search at O(n), they have gained a way of thinking about computational efficiency that will shape every design decision they make for the rest of their career. The emphasis on complexity analysis in CS102 directly anticipates the more rigorous treatment in CS201 (Algorithms) and prepares students for technical interview questions that almost universally involve data structure selection and complexity reasoning.
CS102 also marks the point at which students begin to develop a systems mindset — the ability to think not just about what a program does, but about how it does it, and what the consequences of those implementation choices are at scale. A linked list with a million elements and an array with a million elements may both store the same data, but their performance profiles for insertion, deletion, and access are fundamentally different, and choosing between them incorrectly can mean the difference between a system that scales gracefully to millions of users and one that grinds to a halt. This kind of reasoning — anticipating behavior at scale before the scale is reached — is the hallmark of the software engineering mindset, and CS102 is where it begins.
