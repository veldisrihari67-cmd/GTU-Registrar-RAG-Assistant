GLOBAL TECH UNIVERSITY — COURSE SYLLABUS
CS101: INTRODUCTION TO PROGRAMMING
Credits: 3 | Lecture: 3 hours/week | Lab: See CS101L
Department: Computer Science and Engineering
Prerequisites: None
Co-requisite: CS101L (Programming Lab I)

COURSE DESCRIPTION

CS101: Introduction to Programming is the foundational entry point to the Computer Science curriculum at Global Tech University. This course is designed for students with little to no prior programming experience, though students with informal coding backgrounds will find the structured, theory-grounded approach enriching and clarifying. The course introduces the fundamental concepts of programming using Python, one of the world's most widely used and readable programming languages. Students will learn to think algorithmically, decompose complex problems into manageable components, and express their solutions in clean, correct, and efficient code.
The course covers the essential building blocks of programming: variables and data types, expressions and operators, control flow structures (conditionals and loops), functions and scope, lists and dictionaries, file I/O, and a gentle introduction to object-oriented programming. Equally important, students will be introduced to the discipline of debugging — the art of identifying and fixing errors — and to the practice of writing readable, well-documented code as a form of professional communication.
CS101 is not merely a technical skills course. It is the beginning of a student's journey into computational thinking, a mode of problem-solving that has transformed science, engineering, business, and the humanities. By the end of this course, students will have written dozens of programs of increasing complexity, participated in collaborative coding exercises, and developed the habits of mind — precision, persistence, and creative problem-solving — that distinguish excellent programmers.

LEARNING OBJECTIVES

Upon successful completion of CS101, students will be able to:

Explain the concept of an algorithm and translate informal problem descriptions into step-by-step algorithmic solutions.
Write syntactically correct Python programs that produce expected outputs for a range of input conditions.
Declare and use variables of appropriate data types (integers, floats, strings, booleans) in Python programs.
Apply arithmetic, relational, and logical operators correctly in expressions and conditional tests.
Construct and trace the execution of if, elif, and else conditional statements.
Design and implement for loops and while loops to repeat operations and traverse sequences.
Define and call functions with parameters and return values, demonstrating understanding of local and global scope.
Use Python's built-in list and dictionary data structures to store, access, and manipulate collections of data.
Read from and write to text files using Python's file I/O mechanisms.
Apply string manipulation methods to process and transform textual data.
Identify, categorize, and correct syntax errors, runtime errors, and logic errors using Python's interpreter feedback and systematic debugging strategies.
Write modular programs organized into functions, avoiding code duplication through appropriate abstraction.
Document code using comments and docstrings following standard Python conventions (PEP 257).
Describe and apply basic object-oriented concepts: classes, objects, attributes, and methods.
Test programs systematically using a variety of input cases, including boundary conditions and error cases.
Collaborate on small coding projects using pair programming and code review techniques.
Use a command-line interface and an Integrated Development Environment (IDE) to write, run, and debug Python programs.

WEEKLY COURSE BREAKDOWN

Week 1: Course Introduction and Computational Thinking
Topics: Course overview, what is programming, history of computing, the Python ecosystem, installing Python and VS Code, writing and running your first program ("Hello, World!"), understanding input/output, introduction to the command line.
Assignments: Setup verification lab; Reflection: "Why I want to program."
Week 2: Variables, Data Types, and Expressions
Topics: Variables as named memory locations, Python's dynamic typing, integer and float arithmetic, string literals and concatenation, boolean values, the type() function, operator precedence, common type errors.
Assignments: Problem set 1 — arithmetic expressions and variable manipulation (10 problems).
Week 3: Input, Output, and String Operations
Topics: The input() function, type conversion (int(), float(), str()), f-strings and formatted output, string indexing and slicing, common string methods (upper, lower, strip, split, replace, find).
Assignments: Problem set 2 — string processing programs; Lab: Build a simple user profile program.
Week 4: Conditionals — Making Decisions
Topics: Boolean expressions, comparison operators, if/elif/else syntax, nested conditionals, common pitfalls (using = vs ==), truthiness and falsiness of Python values, short-circuit evaluation.
Assignments: Problem set 3 — decision programs; Quiz 1 (Weeks 1–3 material).
Week 5: Loops — Repetition and Iteration
Topics: The for loop with range(), iterating over strings and lists, the while loop, loop control statements (break, continue, pass), infinite loops and how to avoid them, loop patterns (accumulator, counter, flag).
Assignments: Problem set 4 — loop-based programs; Lab: Build a number-guessing game.
Week 6: Functions — Organizing and Reusing Code
Topics: Why functions matter, defining and calling functions, parameters vs. arguments, return values, void functions, local vs. global scope, the LEGB rule, function composition.
Assignments: Problem set 5 — function design; Midterm Project Announced.
Week 7: Lists and Tuples
Topics: Creating and indexing lists, list methods (append, pop, insert, remove, sort, reverse), list slicing, nested lists, tuples and immutability, list comprehensions (introduction).
Assignments: Problem set 6 — list manipulation; Quiz 2 (Weeks 4–6 material).
Week 8: Midterm Examination and Review
Topics: Comprehensive review of Weeks 1–7 material; Midterm Exam covering all concepts through functions and lists; Midterm Project due (a console-based application of the student's choice, minimum 5 functions, minimum 50 lines).
Assignments: Midterm Exam; Midterm Project submission and code review session.
Week 9: Dictionaries and Sets
Topics: Key-value pairs in dictionaries, creating and accessing dictionaries, dictionary methods (keys, values, items, get, update), iterating over dictionaries, sets and set operations, when to use each data structure.
Assignments: Problem set 7 — dictionary-based programs; Lab: Contact book application.
Week 10: File Input and Output
Topics: Opening and closing files, reading files line by line, writing to files, the with statement (context manager), CSV files and Python's csv module, handling file not found errors.
Assignments: Problem set 8 — file I/O programs; Lab: Simple log analysis tool.
Week 11: Error Handling and Debugging
Topics: Types of errors revisited, Python exceptions, try/except blocks, finally, raising exceptions, building robust programs, systematic debugging strategies, using the VS Code debugger.
Assignments: Problem set 9 — error handling; Debugging challenge lab.
Week 12: Introduction to Object-Oriented Programming (OOP)
Topics: What are objects and classes, defining a class, init method, instance attributes, instance methods, the self parameter, creating objects, encapsulation as a concept.
Assignments: Problem set 10 — basic class definitions; Quiz 3 (Weeks 7–11 material).
Week 13: OOP Continued — Inheritance and Polymorphism
Topics: Inheritance syntax in Python, method overriding, super(), polymorphism through duck typing, when to use OOP vs. functional style, real-world OOP design examples.
Assignments: Problem set 11 — OOP design exercise; Final Project Proposal due.
Week 14: Modules, Libraries, and the Python Ecosystem
Topics: Importing modules, the Python Standard Library (math, random, datetime, os), installing third-party packages with pip, introduction to NumPy and matplotlib (preview), writing your own modules.
Assignments: Library exploration lab; Final Project work session.
Week 15: Final Project Presentations and Course Wrap-Up
Topics: Final project presentations (peer review and instructor evaluation), course retrospective, pathways beyond CS101, what to expect in CS102, software development as a profession.
Assignments: Final Project due (minimum 150 lines, original concept, full documentation); Course Evaluation.

ACADEMIC CONTEXT

CS101: Introduction to Programming occupies a unique and consequential position in the GTU curriculum. It is simultaneously the first step on a technical journey that may span a 40-year career and an immediate, practical encounter with the discipline of problem-solving that defines the computer scientist's craft. The pedagogical philosophy underlying CS101 reflects decades of research in computing education, drawing particularly on the work of constructivist learning theorists who argue that students learn programming most deeply not by watching demonstrations but by writing code, making mistakes, debugging those mistakes, and iterating toward solutions.
Python was selected as the teaching language for CS101 for several well-grounded reasons. Its syntax is clean and English-like, reducing the cognitive overhead that comes with languages like C++ or Java, and allowing students to focus on algorithmic thinking rather than wrestling with semicolons and bracket matching. Python's interactive interpreter provides immediate feedback, making the debugging cycle faster and more instructive. And Python's ubiquity in industry — particularly in data science, artificial intelligence, and web development — means that the skills students acquire in CS101 are immediately applicable in internship and research contexts, unlike skills in more purely academic teaching languages.
The decision to include object-oriented programming in the final weeks of CS101 — rather than reserving it entirely for CS102 or CS190 — is deliberate. Early, gentle exposure to classes and objects allows students to see from the beginning that programming is not just about writing sequences of instructions but about modeling real-world entities, their attributes, and their behaviors. This prepares students conceptually for the more rigorous OOP treatment they will encounter in subsequent courses, reducing the cognitive shock that can occur when entirely new paradigms are introduced without prior context.
CS101 also introduces students to a set of professional habits that will serve them throughout their computing careers. The emphasis on documentation, consistent naming conventions, code review, and testing reflects industry norms that many introductory courses neglect in the rush to cover syntax. At GTU, we believe that a student who writes well-documented, well-tested code from their first week of programming is developing a professional identity, not just learning a language. This identity — as someone who builds things carefully, communicates clearly, and takes responsibility for the quality of their work — is the deepest outcome of CS101.
The course also serves an important gatekeeping and diagnostic function within the CS program. The co-requisite CS101L provides an additional four hours per week of hands-on programming practice in a supervised lab environment, ensuring that students who struggle with the pace of lecture have structured support. The grade of C+ required to proceed to CS102 is not arbitrary — it reflects careful analysis by the GTU CS faculty of the minimum level of programming fluency needed to succeed in a course where algorithms, recursion, and dynamic memory management are introduced at pace. Students who earn below C+ in CS101 are strongly encouraged to repeat the course or to explore alternative pathways, including the Data Science program's more gently scaffolded computing introduction through DS150.
Finally, it is worth contextualizing CS101 within the broader landscape of technology and society. Every student who learns to program gains a form of literacy that is increasingly essential for full participation in the modern economy and civic life. Code shapes elections, medical diagnoses, financial decisions, and social relationships. Students in CS101 are not merely learning a vocational skill — they are gaining access to the mechanisms by which their world is constructed and, eventually, the power to reconstruct it.
