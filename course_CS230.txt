GLOBAL TECH UNIVERSITY — COURSE SYLLABUS
CS230: COMPUTER NETWORKS
Credits: 3 | Lecture: 3 hours/week
Department: Computer Science and Engineering
Prerequisites: CS210 (minimum grade C)

COURSE DESCRIPTION

CS230: Computer Networks provides a rigorous, bottom-up introduction to the architecture, protocols, and design principles of computer networks, from the physical transmission of bits to the application-level protocols that power the internet. The internet is the defining infrastructure of the twenty-first century, and understanding how it works — not as a mysterious cloud of connectivity, but as a precisely designed system of layered protocols, routing algorithms, congestion control mechanisms, and distributed naming systems — is essential for any computer scientist who builds networked applications or reasons about distributed systems.

The course follows the five-layer internet model: physical layer, data link layer, network layer, transport layer, and application layer. At each layer, students learn the problems that layer must solve, the design principles guiding its solutions, and the specific protocols that implement those solutions. Key protocols studied include Ethernet, ARP, IP (IPv4 and IPv6), ICMP, TCP, UDP, DNS, HTTP/HTTPS, SMTP, and TLS. Students complete socket programming assignments in Python, implementing clients and servers that communicate over real network connections.

LEARNING OBJECTIVES

Describe the five-layer internet architecture and explain the principle of layering and encapsulation.
Explain physical layer concepts: signal transmission, modulation, bandwidth, Nyquist and Shannon capacity limits.
Describe data link layer functions: framing, error detection (CRC), and medium access control (CSMA/CD, CSMA/CA).
Explain Ethernet frame structure and the operation of Ethernet switches (learning and forwarding).
Describe the IP protocol: IPv4 datagram format, fragmentation, addressing (CIDR notation, subnetting), NAT.
Explain routing algorithms: link-state routing (Dijkstra), distance-vector routing (Bellman-Ford), and their implementations in OSPF and BGP.
Explain the transport layer's role and contrast the TCP and UDP protocols.
Describe TCP's mechanisms: connection establishment (three-way handshake), reliable data transfer, flow control, and congestion control (slow start, AIMD).
Explain the DNS system: hierarchy, resolution process, caching, and record types.
Explain HTTP/1.1, HTTP/2, and HTTPS protocols and the TLS handshake.
Implement TCP and UDP client-server programs using Python sockets.
Use network diagnostic tools (Wireshark, ping, traceroute, nslookup) to analyze network behavior.
Describe wireless networking (802.11 Wi-Fi) and cellular network architectures.
Explain basic network security threats and defenses: firewalls, VPNs, TLS/SSL.
Reason about network performance metrics: latency, throughput, bandwidth-delay product.

WEEKLY COURSE BREAKDOWN

Week 1: Introduction — The Internet, Protocols, and the Layered Model
Week 2: Application Layer — HTTP, DNS, and Email Protocols
Week 3: Socket Programming — TCP and UDP Clients and Servers
Week 4: Transport Layer — UDP and Principles of Reliable Data Transfer
Week 5: TCP — Reliable Transfer, Flow Control, and Connection Management
Week 6: TCP Congestion Control
Week 7: Network Layer — IP Addressing, Subnetting, and NAT
Week 8: Midterm Examination; Routing Algorithms — Link-State and Distance-Vector
Week 9: Internet Routing Protocols — OSPF, BGP, and the AS Hierarchy
Week 10: Data Link Layer — Error Detection, Multiple Access, and Ethernet
Week 11: Switched Local Area Networks — Ethernet Switches and VLANs
Week 12: Wireless Networks — 802.11 Wi-Fi and Cellular Networks
Week 13: Network Security — Threats, Cryptography Basics, TLS, Firewalls, VPNs
Week 14: IPv6, Software-Defined Networking, and Network Virtualization
Week 15: Content Distribution Networks, P2P Networks, and Course Synthesis; Final Examination

ACADEMIC CONTEXT

The internet is perhaps the most complex artifact that human civilization has ever produced — a global network of billions of devices, connected by millions of miles of fiber-optic cable, wireless links, and satellite connections, coordinated by a suite of protocols that were never formally planned as a unified system but evolved organically over fifty years of research, engineering, and market forces. That this system works — that a packet sent from a device in Vijayawada reliably arrives at a server in San Francisco in under 200 milliseconds — is an engineering miracle that students taking CS230 begin to genuinely appreciate, because they understand the specific mechanisms that make it possible.
The socket programming assignments in CS230 provide a distinctive learning experience: for the first time in the curriculum, students write code that communicates with the real world. Their programs send and receive bytes over actual network connections, and the results are visible and verifiable in ways that purely computational programs are not. This concreteness is motivating and clarifying. When a student implements an HTTP client from scratch and successfully retrieves a web page using only raw socket operations, they have demonstrated to themselves that they understand the protocol at a level that no amount of reading could provide.
The network security content of Week 13 is increasingly essential as security considerations permeate every aspect of software development. Students who understand TLS at the protocol level — the handshake, the certificate chain, the symmetric session keys — are equipped to make informed decisions about the security of the networked systems they build, rather than treating HTTPS as a magic property conferred by a padlock icon in the browser. The rise of network security as a critical professional competency makes CS230's security content not merely interesting but practically urgent.
