GLOBAL TECH UNIVERSITY — COURSE SYLLABUS
CS170: WEB TECHNOLOGIES AND DEVELOPMENT
Credits: 3 | Lecture: 3 hours/week
Department: Computer Science and Engineering
Prerequisites: CS101 (minimum grade C)

COURSE DESCRIPTION

CS170: Web Technologies and Development introduces students to the full stack of technologies that power the modern web, from the foundational languages of the browser (HTML, CSS, JavaScript) through server-side development frameworks and database integration, to deployment on cloud infrastructure. The web is the dominant platform for software delivery in the contemporary world, and an understanding of web technologies — their architecture, their capabilities, and their limitations — is an essential component of a modern CS education regardless of a student's ultimate technical specialization.

The course is structured as a full-stack development journey. Students begin by mastering HTML5 (the language of web content structure), CSS3 (the language of presentation and layout, including Flexbox and CSS Grid), and modern JavaScript (ES6+ features: arrow functions, promises, async/await, modules, destructuring). They then move to the Document Object Model (DOM) and browser APIs, learning to create dynamic, interactive web pages using JavaScript. The back half of the course introduces server-side development using Node.js and Express, RESTful API design and implementation, relational database integration (PostgreSQL), user authentication (JWT and sessions), and deployment to cloud platforms (Heroku or AWS Elastic Beanstalk). The course concludes with an introduction to a modern front-end framework (React), providing students with a preview of the component-based architecture that dominates contemporary front-end development.

Students complete four progressively more ambitious projects: a static personal portfolio website, a dynamic single-page web application with API integration, a full-stack web application with a custom back-end and database, and a final project of the student's own design.

LEARNING OBJECTIVES

Upon successful completion of CS170, students will be able to:

Write semantic, accessible HTML5 documents that correctly structure web content using appropriate elements.
Apply CSS3 for page layout (Flexbox and CSS Grid), typography, color, animation, and responsive design (media queries).
Write modern JavaScript (ES6+) programs demonstrating mastery of variables (let/const), functions, arrays, objects, arrow functions, destructuring, spread/rest operators, and modules.
Manipulate the DOM using the browser's DOM API: selecting elements, modifying content and attributes, handling events, and creating/removing elements dynamically.
Use the Fetch API and XMLHttpRequest to make asynchronous HTTP requests to external APIs, and handle responses using promises and async/await.
Explain the JavaScript event loop, the call stack, the callback queue, and the micro-task queue, and reason about the execution order of asynchronous code.
Implement a Node.js and Express server with RESTful API endpoints supporting CRUD operations.
Design and implement a PostgreSQL database schema, and integrate it with a Node.js application using an ORM (Sequelize) or raw SQL queries (pg library).
Implement user authentication using JSON Web Tokens (JWT) and session-based authentication, including password hashing with bcrypt.
Apply web security best practices: input validation, SQL injection prevention, XSS prevention, CSRF protection, HTTPS.
Build a multi-component React application using functional components, JSX, props, state (useState hook), and side effects (useEffect hook).
Deploy a full-stack web application to a cloud platform and configure environment variables and build processes.
Use browser developer tools for debugging, performance profiling, and network analysis.
Apply version control (Git) and collaborative development practices to web projects.
Evaluate the performance, accessibility, and security of web applications using automated tools (Lighthouse, OWASP ZAP).

ACADEMIC CONTEXT

CS170 addresses a practical reality of the contemporary software development landscape: the web is the universal deployment platform. Whether a student ultimately specializes in machine learning, systems programming, cybersecurity, or database engineering, they will almost certainly work with web technologies at some point in their career — consuming web APIs, building internal tools, or contributing to web-facing products. CS170 ensures that all CS students have a functional, hands-on understanding of the full web development stack, preventing the specialized knowledge gaps that can limit a technically excellent engineer's ability to contribute in cross-functional team environments.

JavaScript, the programming language of the web, deserves special attention as a pedagogical object. It is one of the most widely deployed programming languages in the world and also one of the most idiosyncratic: its dynamic typing, prototype-based inheritance, event-driven concurrency model, and notorious edge cases (type coercion, the this binding, the distinction between null and undefined) make it simultaneously accessible to beginners and treacherous for the unwary. The emphasis in CS170 on understanding JavaScript's execution model — the event loop, the call stack, the distinction between synchronous and asynchronous code — reflects the GTU philosophy that students should understand their tools at a mechanistic level, not merely as APIs to be called.

The security content woven throughout CS170 — SQL injection prevention in Week 10, XSS and CSRF in Week 12, HTTPS and JWT security in Week 11 — reflects the GTU commitment to graduating engineers who build secure systems by default rather than treating security as an afterthought. Web applications are among the most frequently attacked software systems in the world, and the OWASP Top 10 (the ten most critical web application security risks) represents a body of knowledge that every web developer must be familiar with. CS170 ensures that GTU students encounter the OWASP Top 10 not as an abstract list of threats but as concrete vulnerabilities they have implemented, tested, and mitigated in their own applications.
